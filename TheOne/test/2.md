TC-002: Intake Gate Blocks Run When Buyer Role Missing
Purpose

Ensure intake quality gate blocks execution when minimal required fields are missing.

Preconditions

Existing project and scenario created

Test Data

Buyer role: blank

Workaround: Spreadsheets

Trigger event: Hiring new sales reps

Outcome: Reduce lead leakage

Steps

Open Scenario A

Go to intake questions

Answer everything except buyer role

Click Run research and build plan

Expected Results

Run does not start

UI shows blocking message specifying missing requirement: buyer role

inputs.open_questions contains an item prompting buyer role

telemetry.errors remains empty (this is not an error, it is a validation block)

Data Integrity Checks

No new run_id created

No agent timings recorded

inputs.intake_answers contains the provided answers with correct answer_type
