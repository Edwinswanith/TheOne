TC-015: SSE Event Stream Includes Meaningful Intermediate Updates
Purpose

Ensure user does not wait on blank UI and receives continuous updates.

Preconditions

Streaming enabled

Steps

Start run

Subscribe to SSE stream endpoint

Capture events for first 60 seconds

Expected Results

Within first 10 to 20 seconds, receive events: run_started, agent_started evidence_collector, evidence_progress or similar

Evidence results appear before full completion

Events include enough payload to update graph or node drawer incrementally
